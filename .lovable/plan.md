## Goal
Onboard **Harpers Hotshot** as a new client under the Hayes Recruiting organization, mirroring the pattern used for Pemberton, Novco, Day & Ross, James Burg, Danny Herman, and R.E. Garrison.

## Steps

### 1. Database migration — create the client record
Insert a new row in `clients` under Hayes org (`84214b48-7b51-45bc-ad7f-723bcf50466c`):
- `name`: "Harpers Hotshot"
- `organization_id`: Hayes org id
- `status`: 'active'
- Other fields left null for now (can be updated later from the admin UI at `/admin/clients`)

The new client UUID will be generated by `gen_random_uuid()` and captured for use in step 2.

### 2. Register the client in `HAYES_CLIENT_CONFIGS`
Edit `supabase/functions/_shared/hayes-client-handler.ts` to add a new entry keyed `'harpers-hotshot'`:

```ts
'harpers-hotshot': {
  clientId: '<new uuid from step 1>',
  clientName: 'Harpers Hotshot',
  clientSlug: 'harpers-hotshot',
  feedUserCode: 'TBD',         // placeholder until CDL Job Cast code provided
  feedBoard: 'AIRecruiter',
}
```

Because no `feedUserCode` was provided, the entry ships with a `'TBD'` placeholder. Job sync (`?action=jobs`) will return zero jobs until the real code is supplied — application intake (POST) still works.

### 3. Create the dedicated edge function
New file `supabase/functions/hayes-harpers-hotshot-inbound/index.ts` — identical pattern to `hayes-pemberton-inbound`:

```ts
import { serve } from 'https://deno.land/std@0.190.0/http/server.ts';
import { createClientHandler, HAYES_CLIENT_CONFIGS } from '../_shared/hayes-client-handler.ts';

serve(createClientHandler(HAYES_CLIENT_CONFIGS['harpers-hotshot']));
```

### 4. Register in `supabase/config.toml`
Add (next to the other Hayes entries):

```toml
[functions.hayes-harpers-hotshot-inbound]
verify_jwt = false
```

### 5. Verify
- Deploy `hayes-harpers-hotshot-inbound`.
- Curl `GET /functions/v1/hayes-harpers-hotshot-inbound?action=jobs` — should return an empty job list (no feedUserCode yet) without errors.
